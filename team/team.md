# Team

> One `### member:` block per person. `tos setup` fills in the frontmatter
> and the first member; `tos add-member` appends the rest. Humans edit this
> file directly when details change — **agents never write here**.
>
> Grammar (enforced by `tos validate`):
>
> - member id and agent name: lowercase kebab-case, unique across the file
> - `plan:` one of `pro`, `max-5x`, `max-20x`
> - `role:` one of `maintainer`, `member` — at least one maintainer required
>   once any member exists
>
> Example block (keep the `> ` prefix off real entries):
>
> ```
> ### member: alice
> - name: Alice Wong
> - agent: ajax
> - email: alice@example.com
> - plan: max-5x
> - role: maintainer
> ```
